\section{Introduction}
\label{sec:intro}

1.1 Background
With the rapid deployment of Internet of Things (IoT) sensing networks in smart city infrastructures, large volumes of environmental data—such as rainfall, temperature, and humidity—can be continuously collected by geographically distributed sensors. These datasets provide important resources for data-driven urban rainfall prediction and environmental monitoring. However, the distributed nature of IoT data makes traditional centralized machine learning approaches less suitable for real-world deployments, as raw sensor data must be frequently transmitted to cloud servers for model training. This process may introduce privacy concerns and generate substantial communication costs, particularly in bandwidth-constrained and unstable IoT networks.
To address these limitations, distributed learning paradigms such as Federated Learning (FL) and Federated Split Learning (FSL) have been proposed. FL enables multiple clients to collaboratively train a shared model without directly sharing raw data, while FSL further reduces the computational burden on edge devices by splitting the neural network between clients and a central server. In this framework, clients compute only the early layers of the model and transmit intermediate activations to the server, which completes the remaining training process. While this hybrid architecture improves both privacy preservation and computational efficiency, it also introduces new communication challenges in distributed IoT environments.

1.2 Motivation
Despite these advantages, communication overhead remains one of the major challenges in practical FSL deployments for real-world IoT scenarios. In traditional FL systems, communication mainly occurs through model parameter updates exchanged between clients and servers [5]. In contrast, FSL requires the transmission of intermediate activations and gradients during each training step, which can significantly increase communication frequency and data volume.
In real-world IoT deployments, network conditions are often unstable, with issues such as variable latency, packet loss, or sensor failures. These factors further complicate the design of communication-efficient distributed learning systems. As neural networks become deeper and more complex, the size of transmitted activations can grow substantially. In addition, many existing systems treat synchronization intervals as fixed parameters, resulting in frequent communication events that further increase network traffic [6].
These observations suggest that communication efficiency in FSL is influenced by two key factors: the size of transmitted activations and the frequency of synchronization between clients and the server. However, these two dimensions are often studied independently in existing literature, and few studies systematically investigate their combined impact on communication cost, convergence behaviour, and model performance in real-world IoT environments.
Furthermore, rainfall prediction in real-world IoT environments presents additional challenges, including data imbalance caused by the rarity of heavy rainfall events and unstable network conditions among distributed sensors. These factors further motivate the need for communication-efficient and robust distributed learning frameworks.
1.3 Problem Statement
Based on the above observations, the key research problem addressed in this work is how to reduce communication overhead in federated split learning systems while maintaining acceptable rainfall prediction performance.
To address this gap, this work proposes a communication-efficient Federated Split Learning system for rainfall prediction using historical weather data collected from multiple locations around Newcastle upon Tyne through the Open-Meteo Historical Weather API. The proposed system introduces two complementary optimization strategies together with an adaptive control mechanism.
First, a spatial optimization mechanism compresses smashed activations using quantization and Top-k sparsification techniques to reduce transmission size. Second, a temporal optimization mechanism introduces a tunable synchronization interval $/rho$ to regulate the frequency of global model aggregation. In addition, an adaptive scheduler dynamically adjusts these parameters based on runtime network conditions such as latency and bandwidth reported by clients. By jointly controlling communication size and frequency while adapting to network heterogeneity, the framework aims to reduce communication overhead while maintaining stable model convergence.

1.4 Contributions
The main contributions of this work are summarized as follows:
1.A communication-efficient federated split learning framework for rainfall prediction using distributed IoT weather data.
2.A dual optimization strategy that reduces communication overhead through activation compression (quantization and Top-k sparsification) and tunable synchronization intervals.
3.An adaptive scheduling mechanism that dynamically adjusts communication parameters based on runtime network conditions such as latency and bandwidth.
4.A systematic experimental evaluation that analyzes the trade-off between communication efficiency and prediction performance.

1.5 Paper Organization
The remainder of this paper is organised as follows. Section II reviews related work. Section III presents the proposed system design. Section IV describes the experimental setup. Section V discusses the evaluation results. Finally, Section VI concludes the paper.



\section{Related Work}
\label{sec:system}

2. Related Work
This section reviews existing studies related to communication-efficient distributed learning systems for IoT environments. Existing research can be broadly categorised into three directions:
(1) communication-efficient Federated / Split Learning,
(2) adaptive scheduling and system-level optimisation, and
(3) machine learning methods for rainfall prediction in resource-constrained environments.
 
2.1 Communication-efficient Federated / Split Learning
Communication-efficient learning has become an important research topic in distributed machine learning, particularly in IoT and edge environments where bandwidth is limited. In traditional Federated Learning (FL), communication cost is mainly caused by frequent gradient or parameter exchange between clients and the server. To reduce transmission overhead, many studies focus on compression-based methods. For example, compressive sensing–based approaches dynamically adjust compression ratios to reduce bidirectional communication cost [5], while gradient quantisation and sparsification techniques are designed to address bandwidth constraints in wireless edge learning scenarios [6]. These methods show clear improvements in communication efficiency for conventional FL systems.
However, the communication pattern in Federated Split Learning (FSL) differs significantly from standard FL. Instead of exchanging full model gradients, clients send intermediate activations (also called smashed activations) to the server during the forward pass, and gradients are returned during the backward pass. As neural networks become deeper, the size of these activations can increase substantially, leading to high communication overhead. Therefore, compression techniques designed for traditional FL do not always directly address the communication characteristics of FSL.
Recent research attempts to improve efficiency in Split Learning and Federated Split Learning through architectural and synchronisation optimisation. Dynamic split point selection allows the training workload to be adjusted between client and server to reduce communication cost under different conditions [4]. Asynchronous training mechanisms relax strict synchronisation requirements to reduce communication bottlenecks [8]. Other works redesign the learning architecture to reduce server-side state dependency and improve scalability [9;10], while adaptive aggregation strategies aim to balance communication cost and model convergence in distributed environments [11]. Although these methods improve efficiency, they often assume relatively stable network conditions or introduce additional system complexity.
 
2.2 Adaptive Scheduling and System-Level Optimisation
In addition to compression and architectural changes, another research direction focuses on adaptive scheduling and system-level optimisation for distributed learning. These approaches aim to improve performance by considering device capability, network latency, and resource availability during training.
Computation-oriented optimisation methods attempt to reduce the workload on resource-limited client devices. Techniques such as model pruning, adaptive layer reduction, and lightweight network design are widely used in edge learning scenarios. Adaptive pruning strategies dynamically adjust model complexity according to device capability to reduce computation cost [7]. Lightweight prediction models are also designed to balance accuracy and efficiency in resource-constrained environments, particularly for time-series prediction tasks such as rainfall forecasting [2].
However, these approaches mainly optimise computational cost rather than communication overhead. Even with smaller models, clients still participate in frequent synchronisation rounds, which can lead to high cumulative communication cost in distributed systems. In real IoT environments, network latency, bandwidth variation, and device heterogeneity may further increase training time and reduce system stability. Therefore, latency-aware training, resource-aware scheduling, and adaptive communication control are increasingly considered necessary for practical distributed learning systems.

2.3 Machine Learning for Rainfall Prediction in IoT Environments 

Machine learning has been widely applied to rainfall prediction and environmental sensing, especially with the increasing availability of IoT sensor data. Lightweight prediction models are often used to improve efficiency while maintaining acceptable accuracy in resource-constrained environments [2]. These methods demonstrate that machine learning can provide useful prediction capability for real-time monitoring systems. 

However, rainfall prediction remains a challenging task due to data imbalance, temporal variability, and the difficulty of predicting extreme events. Many studies focus primarily on improving modelling accuracy, without fully considering deployment constraints such as communication cost, device capability, and distributed training requirements. In real IoT sensing networks, prediction systems must operate under limited bandwidth, unstable connections, and heterogeneous devices, which makes communication efficiency as important as model accuracy. 

Across the three research directions above, a common limitation can be observed. Most existing studies optimise only one aspect of the system, such as compression, model structure, or distributed architecture, without jointly considering data representation precision, activation size, and synchronisation frequency. As a result, the interaction between communication cost, model convergence, and system scalability remains insufficiently explored, particularly for rainfall prediction systems deployed in real IoT environments. 

To address these limitations, this work proposes a communication-efficient Federated Split Learning framework that jointly optimises activation compression and synchronisation frequency. By integrating spatial compression, temporal communication control, and adaptive scheduling mechanisms, the proposed approach aims to improve the practicality of rainfall prediction systems in real-world IoT sensing networks.

\section{Methodology}
\label{sec:Methodology}
\subsection{Split Learning Architecture}
Our system follows a federated split learning (FSL) architecture for rainfall prediction. Each client performs local sequence encoding, while the server completes prediction from transmitted smashed activations. This design reduces edge-side computation and preserves end-to-end trainability through activation and gradient exchange. For temporal modeling, we use a two-layer LSTM-based split model: the client-side encoder maps each weather window to a fixed-length representation, and the server-side head performs joint rain-occurrence classification and rainfall-amount regression.

\subsubsection{Client-side Encoder}
Each client encodes a 24-step weather window with 5 features into a fixed-length smashed activation and sends only this intermediate representation to the server, instead of raw sequences. This preserves data locality and keeps the transmitted payload stable, which directly supports our communication-oriented compression and adaptive scheduling strategy.

\subsubsection{Server-side Prediction Head}
Beyond the standard split-learning pipeline, we introduce a shared multi-task server head that jointly models rain occurrence (classification) and rainfall amount (regression) from each smashed activation. This design is expected to improve task supervision under rainfall imbalance by activating the regression loss only for rainy samples, while keeping the transmitted cut-layer representation unchanged. Therefore, prediction quality can be improved without increasing per-step communication payload, and the server can still return cut-layer gradients for end-to-end split optimization under centralized supervision.

\subsubsection{Activation and Gradient Transfer}
For each forward step, the client sends compressed activation bytes and metadata (e.g., latency and payload statistics), and the server returns gradient bytes plus control signals for the next step. After local updates, model synchronization is executed every $\rho$ local epochs. Therefore, communication overhead in our FSL system consists of two coupled components: per-step activation/gradient round trips and periodic parameter synchronization, which motivates the adaptive scheduling and compression strategies described in the following subsections.


\subsection{Communication Bottleneck Analysis}
In our federated split learning system, communication cost mainly comes from two parts.
First, in each training step, the client sends smashed activations to the server and receives gradients back. This happens very frequently, so latency and payload size quickly accumulate.
Second, clients still need periodic model synchronization with the server every $\rho$ local epochs, which adds extra upload and download traffic.

Therefore, communication efficiency depends on both data size and communication frequency. In unstable IoT networks, reducing only model complexity is not enough. We need to jointly control activation payloads and synchronization intervals, which motivates our compression and adaptive scheduling design.

\subsection{Adaptive Scheduler}
We design a rule-based adaptive scheduler that adjusts both compression mode and synchronization interval ($\rho$) during training. Clients report latency statistics to the server, and the scheduler applies predefined mapping rules to select the next communication strategy. Due to time and resource constraints, latency is generated by a profiler-based simulator in our experiments, and an exponential moving average (EMA) is used to smooth short-term fluctuations and avoid frequent policy switching. The scheduler changes only communication policies while keeping the model architecture unchanged. Specifically, lower latency keeps lightweight settings (e.g., float32 with smaller $\rho$), while higher latency triggers more aggressive settings (float16/int8/top-k with larger $\rho$). For example, when latency falls in the 4--10 ms range, the scheduler assigns float16 compression and $\rho=2$ (synchronization every two local epochs).


\subsection{Compression Strategies}
We apply activation/gradient compression to both smashed activations and cut-layer gradients using four communication modes: float32, float16, int8, and top-k (implemented as top-k mode). Compression is evaluated in both static and adaptive settings: in static settings, one mode is fixed for the whole run, while in adaptive settings, the scheduler dynamically switches modes according to latency conditions. This design reduces payload size while allowing the system to balance communication efficiency and training stability under varying network conditions.

\section{Experimentl Setup}

\subsection{Dataset Characteristics and Sparsity Analysis}
The experiments are conducted on hourly rainfall-related meteorological observations collected from 12 geographically distributed sensor locations in Newcastle upon Tyne, United Kingdom. The dataset spans from January 2023 to March 2026, and each station provides a continuous time series of local weather conditions. In the current implementation, the model consumes five variables, namely Temperature, Humidity, Pressure, Wind Speed, and Rain.

To preserve the temporal dependency of rainfall dynamics, the data are processed using a sliding-window strategy. For each sample, the model takes the previous 24 hours of multivariate observations as input. The dataset is split chronologically to avoid temporal leakage: the most recent 14 days are used for testing, the preceding 14 days are used for validation, and all earlier observations are used for training. This setting is consistent with realistic forecasting scenarios, where future rainfall must be predicted solely from historical records.

A key characteristic of this dataset is its strong sparsity and class imbalance. According to our statistics, actual rainfall events account for only about 20.8\% of all hourly observations, while the remaining nearly 80\% correspond to dry periods. This highly imbalanced distribution makes rainfall prediction particularly challenging, since a naive model biased toward predicting no-rain can still achieve apparently high accuracy while providing little practical warning value.

For this reason, accuracy alone is insufficient to evaluate system performance. In this work, Recall and F1-score are treated as the primary classification metrics, because they better reflect the model's ability to detect rare rainfall events. In addition, Mean Squared Error (MSE) and Mean Absolute Error (MAE) are used to assess the precision of rainfall amount prediction. This metric design allows us to evaluate both event detection capability and numerical forecasting quality under sparse rainfall conditions.

\subsection{Task Definition and Prediction Framework}
To address the discontinuity and zero inflation caused by sparse rainfall observations, we formulate the prediction problem as a multi-task learning task within the federated split learning framework. The model jointly performs rainfall event classification and rainfall intensity regression from the same historical weather sequence.

The first task is event classification, which determines whether rainfall will occur in the target prediction horizon. Since positive rainfall samples are much rarer than negative samples, the classification branch adopts focal loss to place more training emphasis on difficult and minority cases. In the current configuration, the focal loss uses $\gamma=2.0$ and $\alpha=0.25$, with a classification loss weight of 2.0. A rainfall amount above 0.1 mm is treated as a positive event for the classification target.

The second task is rainfall intensity regression, which estimates the rainfall amount associated with the prediction target. To reduce the influence of skewed rainfall distribution and occasional heavy rainfall peaks, the regression target is transformed using $\log(1+x)$. The regression branch is optimized jointly with the classification branch, with a regression loss weight of 1.0. This design improves learning stability on sparse and long-tailed rainfall data.

To further suppress noisy non-zero outputs during dry periods, the final prediction follows a gated decision rule. Specifically, the system outputs a non-zero rainfall estimate only when the predicted rainfall probability exceeds a threshold of 0.34; otherwise, the prediction is forced to zero. This classification-guided regression strategy is intended to better align the forecasting output with the characteristics of real rainfall events and reduce false alarms in dry environments.

\subsection{System Configuration and Deployment Architecture}
The proposed system is implemented as a prototype federated split learning (FSL) platform. The deployment consists of one central server and three parallel clients, where the data are partitioned by sensor locations to preserve the naturally heterogeneous and non-IID characteristics of geographically distributed weather observations. This setting reflects a realistic IoT sensing scenario in which each edge node only has access to its own local data.

To emulate distributed edge deployment, the system is containerized with Docker and the communication between clients and server is implemented using gRPC. The experiments are conducted on a Windows-based machine with a WSL environment, equipped with 64 GB RAM and an NVIDIA RTX 4090 Laptop GPU. This environment enables reproducible distributed execution while maintaining sufficient compute capacity for repeated experimental runs.

The model itself follows a split architecture. On the client side, a two-layer LSTM encoder is used to process the input sequence, with hidden size 64, dropout 0.3, and sequence length 24. On the server side, a lightweight prediction head with width 64 and dropout 0.1 performs the joint classification and regression tasks. The training process uses a batch size of 32, learning rate 0.0005, 30 global rounds, and 10 local steps per client before synchronization decisions are applied.

In addition to the distributed learning pipeline, the system includes a latency generator to simulate heterogeneous IoT communication conditions. The generator injects a configurable base latency, random jitter, and client-specific offset, thereby reproducing the variability commonly observed in real-world edge networks. This component is essential for evaluating whether the proposed communication-efficient strategies remain effective under unstable network conditions.

\subsubsection*{Parameter Definitions}
\begin{itemize}
  \item \textbf{Latency Config}: Parameters $(L_{base}, L_{offset}, L_{jitter})$ used to simulate network conditions.
  \item \textbf{Dynamic Compression}: Automatic switching between Float32, Float16, INT8, and Top-K based on real-time network pressure.
  \item \textbf{Dynamic Sync ($\rho$)}: Represents the number of local steps before global weight synchronization, scaling from 1 up to 20 in extreme cases.
  \item \textbf{Burst Latency}: A transient 20ms surge added every 50 steps to simulate peak congestion.
\end{itemize}

\subsubsection*{Limitations and Scope}
Due to constraints in time and computational resources, this setup cannot encompass all real-world edge cases. However, these 14 scenarios represent the most common challenges in global IoT deployments, providing a scientifically sound foundation for evaluating the system's adaptive robustness.
