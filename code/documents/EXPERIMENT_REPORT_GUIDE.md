## 1. Detailed Model Architecture
The system employs a physically partitioned architecture to separate computation and protect data:

### A. Edge Model (Client LSTM)
- **Input**: $24 \times 5$ Tensor (Temperature, Humidity, Pressure, Wind Speed, Rain for the past 24 hours).
- **Core**: Double-layer LSTM (Hidden Size: 64, num_layers: 2).
- **Dropout**: 0.3 (to enhance generalization).
- **Output**: 64-dimensional **Smashed Activation** (final hidden state $h_n$ of the sequence).

### B. Cloud Model (Server Head)
- **Input**: Receives 64-dimensional activations from clients.
- **MLP Structure**: 
    - Linear(64 → 32)
    - **LeakyReLU(0.1)** (Activation function)
    - Linear(32 → 1)
- **Final Activation**: ReLU (to ensure predicted rainfall $\ge 0$).
- **Output**: Predicted total rainfall for the next 3 hours (mm).

## 2. Core Sections for the Experiment Report

### Section 1: Data Engineering
- **Data Source**: Open-Meteo historical weather data (Newcastle region).
- **Balanced Sampling Strategy**: Describe how the **50/50 Balanced Sampling** addresses data sparsity and prevents the "always guessing zero" bias.
- **Standardization**: Detail the **Dynamic Z-score** implementation and how it automatically normalizes features per sensor group.

### Section 2: Training Configuration
- **Optimizer**: Adam (Learning Rate = 0.0005).
- **Gradient Management**: Explain the use of `clip_grad_norm_ (1.0)` to prevent gradient explosion.
- **Early Stopping**: Record the threshold for **Early Stopping** (Patience = 20 rounds).
- **Aggregation**: Weights are synchronized using the FedAvg (Federated Averaging) algorithm.

### Section 3: Evaluation Metrics
The report should include the following data generated by `run_evaluation.py`:
- **MSE (Mean Squared Error)**: Measures numerical prediction precision.
- **Rain/Dry Accuracy**: Classification accuracy for detecting rain vs. no-rain (threshold set at 0.1mm).
- **Sensor Consistency**: Comparison of performance across the 12 distinct geographical sensors.

### Section 4: Communication Analysis (Optional)
- **Transport Efficiency**: Discussion on the data payload size of Smashed Activations.
- **Latency Performance**: Average Latency (ms) for a full Forward/Backward cycle.
